# 코드리뷰
## 코딩 컨벤션
- 프로그래밍 스타일에 대한 가이드라인 또는 규칙
- 다른 개발자가 작성한 코드를 조금 더 쉽게 읽을 수 있도록, 코드의 복잡도가 높아지지 않도록 정하는 것
- 가독성 좋고, 일관성 높은 작업을 위해
- 참고 자료
    - Intelli J
    - Naver Convention(naver.github.io)
        - Formatter 설정 파일 다운
    - Google Convention
        - Formatter 설정 파일 다운
        - black tool을 사용하여 다양한 python IDE 에서 formatter 사용 가능
        - Google code style formatter 사용
    - Javascript Formatter
        - ESLint 와 Prettier 사용
                - 주의사항: 규칙 충돌 방지하는 거 설치해야됨

## 코드리뷰
- 소프트웨어를 실행하지 않고 잠재적 결함을 찾아내고 개선해 소프트웨어의 품질을 높이고자 하는 활동
- 개발자가 작성한 코드를 검토
- 규칙 1
    - 코딩컨벤션에 따라 클린하고 일관되게 구현
    - 왜 개선이 필요한지 이유를 충분히 설명
    - 답을 알려주기보다는 스스로 고민하고 개선 방법을 선택 할 수 있도록 리뷰
    - 모두를 배려하는 마음으로 진행

- 고려사항
    - 리뷰어의 시간을 아낄 수 있도록 고려
    - 완벽함 대신 지속적인 개선을 목표로 코드의 유지 보수성과 가독성을 개선하는데 초점을 맞춰 코드리뷰 진행
    - 코드리뷰로 효과가 바로 나타나지 않을 수 있으므로, 인내심을 갖고 꾸준하게 진행
