# Container
- 소프트웨어 서비스를 실행하는 데 필요한 특정 버전의 프로그래밍 언어 런타임 및 라이브러리와 같은 종속 항목과 애플리케이션 코드를 함께 포함하는 경량 패키지

## 리눅스 컨테이너 (LXC)
- 리눅스 커널을 공유하면서 프로세스를 격리된 환경에서 실행하는 기술
- namespace와 cgroups 등의 커널 기능을 활용해 격리되어 실행

## namespace
- 시스템에서 독립된 공간을 격리된 환경에서 운영하는 가상화 기술
- 프로세스가 시스템 리소스를 독립적으로 관리할 수 있음
- 똑같은 컨테이너를 복사해서 사용가능하다
- 종류
    - PID(Process ID)
    - MNT(Mount)
    - NET(Network)
    - IPC(Inter-Process Communication)
        - 프로세스간 통신 리소스 격리
    - UTS(UNIX Time Sharing)
        - 시스템 호스트 이름과 도메인 이름 격리
    - USER
        - 사용자 ID와 그룹 ID를 격리

## cgroups
- 자원 사용을 제한
- 같은 호스트에서 동작하는 서로 다른 컨테이너에 영향을 주지 않도록 막아주는 역할

## Docker
- 서비스를 패키징하고 배포하는데 유용한 오픈소스 프로그램
- 컨테이너에 프로세스 단위로 격리할 수 있는 엔진으로 이루어짐
- 사용 이유
    - 서버 확장할 때 마다 매번 환경을 새로 구축 할 필요 없음
    - 관리 비용 절감
    - 컨테이너 복사만으로 환경 구축 가능

- 도커 레이어
    - 수정된 값만을 다운받고 관리할 수 있게 레이어라는 개념을 사용하여 경량화
    - 도커 이미지는 여러 개의 읽기전용 레이어로 구성
    - 파일이 추가되거나 수정되면 기존 레이어는 변하지 않고 그 위에 새로운 레이어가 생성됨

- 도커 컨테이너
    - 컨테이너는 종료되어도 메모리에서 삭제되지 않고 남아있어 재가동 가능

- 도커 네트워크
    - 가상 브리지 네트워크 장치와 라우팅 테이블 생성
    - 도커는 내부적으로 리눅스 브리지(veth pair)를 사용
    - iptables 규칙을 생성

- 도커 네트워크 구조
    - bridge
        - 하나의 호스트에서 여러 컨테이너들이 서로 소통가능
    - host
        - 호스트와 동일한 네트워크 환경으로 컨테이너를 사용
    - none
        - 컨테이너 네트워킹을 불가능하게 만듦
    - overlay
        - 여러 호스트에 분산되어 돌아가는 컨테이너들 간에 네트워킹을 위해서 사용

- 도커 Inspect
    - 도커 개체에 대한 low level 정보 반환
    - 도커에 의해 제어되는 구문

- 도커 Prune 명령
    - 사용하지 않는 컨테이너, 이미지, 볼륨, 네트워크를 일괄 삭제하는 명령
        - docker[옵션]prune -f
        - docker system prune
        - docker image prune
        - 등등

- 도커 디스크 용량 확보
    - 도커 이미지를 빌드하게 되면 중첩되는 레이어는 다시 빌드 하지 않기 
    - 로컬 시스템의 모든 docker 컨테이너 삭제
        - docker rm -f $
    - 로컬 시스템의 모든 docker 이미지 삭제
        - docker rmi $
