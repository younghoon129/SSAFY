# 최적화
- 필요한 이유
    - 페이지 로딩 시간이 1초 지연될 때마다 사용자의 약 10%가 사이트를 떠난다

## Core Web Vitals
- 프론트엔드 성능은 사용자의 경험과 웹사이트 성공에 매우 중요함
- 구글이 웹사이트 성능 최적화를 위해 세운 전략?

- LCP
    - 웹 페이지를 로드 할 때 페이지의 가장 큰 콘텐츠 요소가 브라우저 화면에서 얼마나 빨리 렌더링 되는지 측정하는 지표
    - 페이지가 전부 노출되는데 2.5초(좋음) - 보통 - 4초 이상(나쁨)
- INP
    - 웹 페이지의 전반적인 응답성을 평가하기 위한 측정 항목
    - 클릭, 탭, 키보드 상호작용의 지연시간을 관찰하여 가장 긴 지연 시간을 최종 평가
    - 200 ms(좋음) - 보통 500 ms 이상(나쁨)
- CLS
    - 0.1 - 0.25
- FID
    - 브라우저 반응 시작되기까지 걸리는 시간 측정 지표
    - 100 - 300

- Lighthouse
    - 구글에서 개발한 오픈소스르로 웹 페이지 품질을 개선할 수 있는 자동화 툴
    - 사이트를 검사하여 성능 측정 가능
    - 성능, 접근성, PWA 등 체크 가능

- 분석 결과 항목
    - performance
    - accessful
    - seo
    - pwa

- PageSpeed Insights
    - Google 에서 제공하는 무료 웹 성능 테스트 도구

- 크롬 개발자 도구
    - 웹 개발 및 디버깅에 필요한 다양한 기능 제공하는 도구 모음
    - Network 탭에서 로딩되는 리소스들의 정보 확인
    - Waterfall 차드에서 로딩 시간과 병목 지점 발생하는 부분 알 수 있음
