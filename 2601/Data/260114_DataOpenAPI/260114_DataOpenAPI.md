## OpenAPI
- OpenAPI는 누구나 사용할 수 있도록 공개된 API를 말하며, 웹 사이트가 가진 기능을 모두가 이용할 수 있도록 공개한 프로그래밍 인터페이스
- 많은 서비스 업체 및 공공 기관에서 제공하는 서비스를 외부에서 사용할 수 있게 API를 제공하는 형태
- 서버 -> USER -> 서비스, 애플리케이션, 플랫폼
- 공개된 OpenAPI 사용하기
    - 만약 기관 등의 서버에서 Open API를 제공하고 있다면, 손쉽게 정제된 형태의 데이터를 가져올 수 있음
    - 주로 XML이나 JSON의 데이터 형식으로 인코딩 있음
- XML
    - 각 자료마다 태그를 붙여서 자료의 형태 표현
    - 트리 형태의 구조
    - EmployeeData -> employee id -> firstName, lastName 등
- CSV
    - ',' 로 자료 내용 구분, 정형 데이터
    - 자식 개념 아님, 통계데이터 및 목록 같은 데이터
- JSON
    - 키 : 값 페어로 데이터 표현(파이썬 딕셔너리)
    - 구조 명확, 읽기 쉬움
- HTTP 통신
    - 웹 브라우저와 웹 서버 사이에 데이터를 주고 받는데 사용되는 통신
    - Request(요청)
    - Response(응답)
- GET 메소드와 URL
    - GET 메소드
        - 웹 서버에게 파라미터를 포함해 요청을 보내는 가장 쉬운 방법, URL에 파라미터 정보를 담아서 보내는 것
        - 요청: 사용자가 서버에게 요구하는 것
        - 응답: 서버가 요청에 대한 대답을 담아 HTML문서로 주는 것

## 공공데이터 포털 활용
- 공공 데이터 포털의 OpenAPI 사용 절차
    - 사용자의 API key 사용신청
    - 공공데이터 포털이 제공기관에 OpenAPI 호출
    - 제공기관이 API 회신
    - 공공데이터 포털이 데이터(XML, JSON, CSV) 확인
    - 사용자에게 API 결과 응답
    - 사용자는 파싱, 분석 및 저장
- 공공 데이터 포털의 OpenAPI 활용
    - 요청변수에 무엇을 입력해야 하는지 설명되어있음
    - 샘플데이터의 입력방식을 참고하여 GET 요청 보냄
    - 필수 항목(ServiceKey)은 반드시 입력

## 기업 OpenAPI
- 기업 OpenAPI 활용
    - 공공데이터는 정형, 정적 데이터 위주
    - 기업 API는 실시간 / 서비스 중심 데이터 등을 다양하게 제공
    - 다양한 기업에서 OpenAPI 제공
- Naver
    - 로그인 방식의 OpenAPI(Oauth 기반)
        - 네이버 로그인 인증 필요(OAuth)
        - 사용자 권한을 위임받아 기능 수행하는 것이 주 목적
    - 비로그인 방식 OpenAPI(API key 기반)
        - Client ID / Client Secret으로 인증
        - 사용자 로그인 불필요
        - 데이터 조회 중심
