# DBMS_정규화
- 관계형 데이터베이스의 설계에서 중복을 최소화하게 데이터를 구조화하는 프로세스
- 이상 현상을 방지하기 위해 데이터베이스를 올바르게 설계해 나가는 과정
    - 이상 현상: 데이터 삽입, 수정, 삭제, 연산 할 때 발생할 수 있는 부작용

- 이상 현상
    - 삽입 이상
        - 불필요한 데이터도 함께 삽입해야 하는 문제
    - 갱신 이상
        - 데이터가 불일치하게 되는 문제
    - 삭제 이상
        - 꼭 필요한 데이터까지 같이 삭제되는 데이터 손실 문제

- DBMS Key
    - 기본키
        - NULL 값, 중복 없어야 됨
        - 복합키 피하고 테이블에 기본키 하나만 있을 것
    - 외래키
        - 데이터의 무결성을 보장하기 위해 설정
    - 복합키
        - 여러 열을 조합하여 기본키 역할을 할 수 있게 만든 키

- 제 1 정규화
    - 각 컬럼은 하나의 종류, 값만 가지자
    - Telephone Number1, Telephone Number2 이런 식으로 저장하지 말자

- 제 2 정규화
    - 제 1 정규화를 만족해야 함
    - 주로 복합 키를 가진 테이블에서 발생함
        - 데이터 중복성 문제 발생할 수 있음
    - 부분 종속성
    - 복합키 아니면 제 2 정규화는 자동으로 성립

- 제 3 정규화
    - 기본키를 제외한 속성들 간의 이행 종속성이 없어야 함
    - 기본키가 단일키이지만 반복적 데이터가 많은 경우 필요

- 역정규화
    - 경험해보면 좋음

- 버퍼 오버플로우 공격
    - 데이터의 길이에 대한 불명확한 정의를 악용한 덮어쓰기
    - 저장할 때 버퍼의 용량을 초과하여 데이터를 입력하면 발생
    - 시스템에 악성 코드가 실행될 수 있도록 설계
        - 입력 받는 폼은 모두 글자수 체크 할 것
            - 로그인 창에 많은 인수 값 전달 시 에러/오류가 발생하는지 점검
            - 회원 정보 입력에 올바르지 않은 값이나 너무 긴 문자 전달 시 에러나 오류가 발생하는 지 점검
            - 검색인 게시물 작성시 허용된 글자수 보다 많은 글자가 복사되어 입력되는 경우 오류가 발생하는 지 점검
            - 값 설정할 때 음수처리 어떻게 할 건지 고민할 것
    
- 버퍼 오버플로우 발생 사례
    - MySQL Authentication 버퍼 오버플로우
        - MySQL/MariaDB에서 특수한 패킷을 조작해 스택 버퍼 오버플로우 발생
        - 공격자는 루트 권한으로 임의 코드 실행 가능
        - 주로 MySQL의 네트워크 핸들러(mysqld)에서 발생
    - PostgreSQL contrib/intarray 모듈 버퍼 오버플로우
        - intarray 확장 모듈의 문자열 처리 함수에서 길이 검사 누락
        - 공격자가 긴 문자열을 입력하면 스택 오버플로우 -> RCE 가능
    - Oracle DB TNS Listener 버퍼 오버플로우
        - TNS Listener
    - SQL

- 보안 설저 주의 사항
    - 안정성이 검증된 최신 버전으로 패치
    - 필요한 크기만큼만 받을 수 있도록 개발
    - 잘못된 인수 값이 에러 페이지를 반환하지 않도록 설정
    - 동적 메모리 할당 시, 그 값이 음수가 아닌지 검사하여 버퍼 오버플로우 예방
    - 한국인터넷진흥원(사이버 보안 취약점 정보 포털)
    - CWE(일반적인 소프트웨어와 하드웨어의 보안 약점을 나열한 공식적인 리스트)
