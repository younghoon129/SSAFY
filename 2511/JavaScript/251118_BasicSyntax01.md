# Basic Syntax01


## 데이터 타입
- 원시 자료형
    - 값 자체가 변수에 직접 저장
    - 불변, 변수 간 할당 시 값이 복사
    - 종류
        - Number
            - 문자열 + 숫자 시 숫자가 문자열 자동 형 변환
            - 정수, 실수 구분 x 
        - String
            - +로 문자끼리 결합
            - `${}` -> Fstring (ES6+ 부터 지원)
        - Boolean
            - Boolean 데이터 타입 아니어도 '자동 형변환' 으로 인해 true 또는 false로 변환됨
        - null
            - '의도적으로 값이 없음'을 나타냄
            - 타입: Object -> 0
        - undefined
            - '값이 할당되지 않음' 의도 x
            - 타입: undefined -> NaN
- 참조 자료형
    - 데이터가 저장된 메모리의 주소가 변수에 저장되는 자료형
    - 가변, 변수 간 할당 시 주소가 복사
    - Objects(Object, Array, Function)

## 연산자
- 종류
    - 할당 연산자
        - 단축 연산자 지원(-=, += 등)
    - 증가, 감소 연산자
        - 증가: ++
        - 감소: --
        - 가독성을 위해 단축 연산자 권장
    - 비교 연산자
        - 결과값 boolean으로 반환하는 연산자(>, <)
    - 동등 연산자(==)
        - boolean 값을 반환
        - 타입 상관 x (암묵적 타입 변환)
        - 같은 객체인지 확인
    - 일치 연산자(===)
        - 객체, 값, 타입 확인
        - 암묵적 타입 변환 X
    - 논리 연산자
        - and(&&)
        - or(||)
        - not(!)
        - 단축평가 지원

## 조건문
- if
  - 삼항 연산자
    - 복잡한 로직, 대다수의 경우 가독성 떨어질 수 있음
    - (조건) ? (참일 경우 반환 값) : (거짓일 경우 반환 값)
  
## 반복문
- while
- for
  - i 를 재할당 하면서 사용하기 때문에 const 사용하면 에러
- for in
  - 키에 대한 반복
  - 배열에 적용할 경우 인덱스를 문자열로 반환
  - const 사용해도 에러 발생안함
- for of
  - 값에 대해 반복
  - const 사용해도 에러 발생안함

## 함수
- 선언식
  - 호이스팅 됨
  - 코드의 구조, 가독성 장점
- 표현식
  - 호이스팅 안됨
  - 익명함수 사용 가능
- 표현식 사용 권장하는 이유
  - 예측 가능성
    - 호이스팅 영향 없으니 더 명확하게 예측 가능
  - 유연성
    - 변수에 할당되므로 함수를 값으로 다루기 쉬움
  - 스코프 관리
    - 블록 스코프를 가지는 let이나 const와 함께 사용하여 더 엄격한 스코프 관리 가능

## 매개변수
- 기본 함수 매개 변수
  - 함수 호출 시 인자를 전달하지 않거나 undefined를 전달할 경우, 지정된 기본값으로 매개변수를 초기화
- 나머지 매개 변수
  - 정해지지 않은 개수의 인자들을 배열로 모아서 받는 방법
  - 작성 규칙
    - 함수 정의 시 나머지 매개변수는 하나만 작성 가능
    - 나머지 매개변수는 함수 정의에서 매개변수 마지막에 위치해야 함
    - '...'
- 매개변수와 인자 개수 불일치 할 경우
  - 매개변수 > 인자 개수
    - 누락된 인자는 undefined로 할당
  - 매개변수 < 인자 개수
    - 초과 입력한 인자 사용 안함
  
## Spread syntax
- 전개구문('...')
- 배열이나 문자열처럼 반복 가능한 항목들을 개별 요소로 펼치는 것
- 전개 구문 활용처
  - 함수와의 사용
    - 함수 호출 시 인자 확장, 나머지 매개변수
  - 객체와의 사용
  - 배열과의 활용

## 화살표 함수 표현식
- function 키워드 제거, 매개변수와 중괄호 사이 화살표 작성
- 함수의 매개변수가 하나 뿐이라면, 매개변수의 () 제거 가능
- 함수 본문의 표현식이 한 줄이라면, {}와 return 제거 가능
- 인자 없으면 () 대신 _ 사용 가능

## NaN
- NaN을 반환하는 경우 예시
  - 숫자로서 읽을 수 없음
  - 결과가 허수인 수학 계산식
  - 피연산자가 NaN
  - 정의할 수 없는 계산식
  - 문자열을 포함하면서 덧셈이 아닌 계산식
  - NaN === NaN -> False, type(NaN) -> number
