# controlling event
- event
    - 웹에서의 거의 모든 상호작용은 이벤트와 함께 함
    - 화면 스크롤
    - 버튼을 클릭했을 때 팝업 창 출력
    - 마우스 커서의 위치에 따라 드래그 앤 드롭
    - 사용자의 키보드 입력 값에 따라 새로운 요소를 생성
    - 이벤트, 리스너, 핸들러
- event object
    - DOM 에서 이벤트 발생하면, 브라우저는 해당 이벤트에 관한 정보를 담은 'event object' 생성
    - 종류
        - mouse
        - input
        - keyboard
- 과정
    1. 요소 찾기
    2. 이벤트 핸들러 작성
    3. 연결
- event handler
    - 특정 이벤트가 발생했을 때 실행되는 (콜백) 함수
    - .addEventListener로 등록 
- .addEventListener('type', handler)
    - 핸들러를 DOM 요소에 연결하는 역할
    - 특정 DOM 요소에, 지정한 이벤트가 발생했을 때 실행할 이벤트 핸들러를 등록하는 메서드
    - DOM 요소: HTML 문서의 각 태그를 하나의 객체로 변환하는 것
    - 수신할 이벤트: 무언가 일어났다는 신호 또는 사건
    - 핸들러: 특정 이벤트가 발생했을 때 실행되는 (콜백) 함수
    - type: 수신할 이벤트 유형
    - handler: 자동으로 event 객체를 첫번째 매개변수로 받음

- 버블링
    - 한 요소에 이벤트가 발생하면, 해당 요소의 핸들러가 동작한 후 이어서 부모 요소의 핸들러가 동작하는 현상
    - 가장 최상단의 조상 요소(document)를 만날 때까지 이 과정이 반복되면서 요소 각각에 할단된 핸들러 동작
    - 중요성
        - 각자 다른 동작을 수행하는 버튼이 여러 개가 있다고 가정하면, 각 버튼마다 서로 다른 이벤트 핸들러를 등록해야할까?
        - 요소들의 공통 조상에 이벤트 핸들러를 하나만 등록하면, 여러 자식 요소에서 발생하는 이벤트를 한곳에서 효율적으로 다룰 수 있음
        - 공통 조상에 할당한 핸들러에서 event.target을 이용하면 실제 어떤 버튼에서 이벤트가 발생했는지 알 수 있기때문
- 이벤트가 정확히 어디서 발생했는지 접근할 수 있는 방법   
        - event.currenTaget
        - '현재' 요소
        - 항상 이벤트 핸들러가 연결된 요소만을 참조하는 속성
        - this와 같음
    - event.target
        - 이벤트가 발생한 가장 안쪽의 요소(target)를 참조하는 속성
        - 실제 이벤트가 시작된 요소
        - 버블링이 진행 되어도 변하지 않음
        - 다시보기

- 캡처링
    - 이벤트가 최상위 조상에서 타겟 요소까지 하위로 전파되는 단계(버블링과 반대)
    - table의 하위 요소 td를 클릭하면 이벤트는 먼저 **최상위 요소부터 아래로 전파**
    - 실제 이벤트가 발생한 지점에서 실행된 후 **다시 위로 전파**(버블링)
- currentTarget 주의사항
    - console.log()로 event 객체를 출력할 경우, currentTarget 키의 값은 null 임
    - currentTarget은 이벤트가 처리되는 동안에만 사용가능하기 때문
    - 대신 console.log(event.currentTarget)로 확인은 가능함
    - currentTarget 이후의 속성 값들은 'target'을 참고해서 사용하기
- event 활용
    - .trim : 입력값 비어있는 지 확인 (비어있지 않으면 true)
    - alert : 오류창? 
    - lodash : 모듈성, 성능 및 추가 기능 제공하는 유틸리티 라이브러리(간편하고 유용한 함수들 제공)
- 이벤트 기본 동작 취소하기
    - HTML의 각 요소가 기본적으로 가지고 있는 이벤트가 방해 되는 경우가 있음
        - 이벤트의 기본 동작 취소의 필요성
    - 예시
        - form 요소의 제출 이벤트를 취소 -> 페이지 새로고침 방지
        - <a> 태그를 클릭했을 때 페이지 이동 방지, 다른 로직을 대신 수행하게 할 수 있음
    - .preventDefault()
        - 해당 이벤트에 대한 기본 동작을 실행하지 않도록 지정
        - 예시: 회원가입 정보 입력하다 실수로 뒤로가기 눌렀을 때 '정말 이동하시겠습니가?' 라는 문구 생성
- addEventListener와 화살표 함수 관계
    - 화살표 함수는 자신만의 this를 가지지 않음
    - 자신이 선언된 상위 스코프의 this를 그대로 물려받아 사용
    - 이벤트 핸들러로 화살표 함수를 사용하면, this는 대부분 전역 객체를 가르킴
    - 해결책
        - 일반 함수로 사용하기
        - 화살표 함수일 경우 event.currentTarget을 사용하기
