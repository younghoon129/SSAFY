# BasicSyntax02

- Object (파이썬에서 dictionary)
    - 객체 구조
    - key는 문자형만 가능
    - value는 모든 자료형 가능
- 속성 참조
    - ',' 표기법 -> . 뒤에 key 이름만 가져옴(key 이름 하나 인거지)
    - [] 표기법 -> key 이름에 띄어쓰기 있으면 무조건 [], for와 같이 써서 여러개 키를 부를 수 있음
- in 연산자
    - 프로토타입 체인 속성 있음(자신에게 없는 속성, 기능을 부모, 조상 순으로 찾음)
        - 의도치 않은 true 발생
    - 객체에서 값의 포함 여부를 확인할 경우
        - in 대신 hasOwnProperty() 메서드 사용할 것
- Method
    - 객체 속성에 정의된 함수
    - this 키워드를 이용해 객체 자신의 속성이나 메서드에 접근
- this 
    - 함수나 메서드를 호출한 객체를 가리키는 키워드
    - 일반 함수에서 단순 호출
        - 전역 객체
    - 객체에서 메서드 호출
        - 메서드를 호출한 객체
    - forEach 는 일반 함수로 호출 -> this가 전역 객체 가르킴
        - 해결하기 위해 화살표 함수(=>) 이용
        - 자신만의 this를 가지지 않고, 외부 함수에서 this 값 가져옴

## 추가 객체 문법
- 단축 속성
    - 키 이름 값, 변수 이름 같은 경우 단축 구문 사용 가능
- 단축 메서드
    - 메서드 선언 시 function 키워드 생략 가능
- 계산된 속성
    - 키가 대괄호([])로 둘러 싸여 있는 속성
    - 고정된 값이 아닌 변수 값 사용 가능
    - a = 'my', b = 'me' -> [a + b] -> myme
        - 이 경우 key 값이 중복될 수 있어 조심해야 됨
- 구조 분해 할당
    - 배열 또는 객체를 분해, 객체 속성을 변수에 쉽게 할당할 수 있는 문법
    - const userInfo 에 있는 Object들을 const { key 명 } = userInfo 하면 알아서 할당 가능
- 객체와 전개 구문
    - 객체 복사
    - 얕은 복사 활용 가능
- 객체 메서드
    - Object.keys()
    - Object.values()
    - Object.entries()
- Optional chaining(?.)
    - 속성이 없는 중첩 객체에 접근하려 할 때 에러 발생 없이 안전하게 접근
    - 참조 대상이 null or undefined라면 에러가 발생하는 것 대신 평가 멈추고 undefined 반환
    - user.address = undefined 인데 user.address.street 하면 에러 발생
    - 만약 undefined 가 아니고 객체 있으면 객체 값을 가져옴
    - 남용해선 안됨. 존재하지 않아도 괜찮은 대상에만 사용해야 함
    - 앞의 변수는 반드시 선언되어 있어야 함.
- JSON
    - 일정한 형식을 가진 '문자열'
    - key-Value 형태로 이루어진 자료 표기법
    - JSON.parse()를 사용해 문자열을 객체로 변환(Object로)
    - JSON.stringfy()를 사용해 객체를 문자열로 변환(JSON으로)

## 배열
- 순서가 있는 데이터 집합을 저장하는 자료구조
- 객체는 key로 데이터를 관리, 순서 중요하지 않음
- 단점: 특정 값을 찾기 위해서는 배열의 처음부터 순서대로 확인해야 함
- 메서드
    - push()
        - 배열 끝에 요소 추가
        - 반환 값: 새로운 배열의 길이
    - pop()
        - 배열 끝 요소 제거
        - 반환 값: 제거한 요소
    - unshift()
        - 배열 앞에 요소 추가
        - 반환 값: 새로운 배열의 길이
    - shift()
        - 앞 요소 제거
        - 반환 값: 제거한 요소

- 다른 함수에 인자로 전달되는 함수
- 외부 함수 내에서 호출되어 작업 진행
- 재사용할 수 있어 유용함

## Array Helper Methods
- 배열 조작을 보다 쉽게 수행할 수 있는 특별한 메서드 모음
- ES6에 도입
- 배열의 각 요소를 순회하며 각 요소에 대해 함수(콜백함수)를 호출
- 대표 메서드
    - **forEach()**
        - python 의 map 느낌
        - 배열 내의 모든 요소 각각에 대햏 함수(콜백함수)를 호출
        - 반환 값 X
        - 목적: 탐색
    - **map()**
        - python 의 map 느낌
        - 배열 내의 모든 요소 각각에 대해 함수(콜백함수)를 호출
        - 함수 호출 결과를 모아 새로운 배열 반환
        - 목적: 탐색 후 재가공
    - **filter()**
    - every()
    - some()
    - **reduce()** (중요)
    - 등

- 메서드 호출 시 인자로 함수(콜백함수)를 받는 것이 특징

## forEach()
- 배열 내의 모든 요소 각각에 대해 함수(콜백함수)를 호출
- 3가지 매개변수로 구성
    - arr.forEach(function (item, index, array)){
        // do something
    }
    - item: 처리할 배열의 요소
    - index: 처리할 배열 요소의 인덱스(선택 인자)
    - array: forEach를 호출한 배열(선택 인자)
- 반환 값: undefined
- break로 중단할 수 없음
- 화살표 함수 사용 권장 

## map()
- 배열 내의 모든 요소 각각에 대해 함수(콜백함수)를 호출
- forEach()와 매개 변수 동일
- 반환 값: callback의 결과를 모은 새로운 배열
- 새로운 배열 반환하기 때문에 다른 메서드를 체이닝할 수 있음
- 원본 배열 수정 X

## 콜백 함수의 이점
- 함수 유연성
    - 함수 호출 코드에서 콜백 함수의 동작을 자유롭게 변경 가능
    - map 함수는 동일하지만, 어떤 '콜백 함수'를 전달하느냐에 따라 결과 달라짐
- 비동기적 측면
    - setTimeout 함수는 일정 시간이 지난 후에 실행됨

## forEach에서 break 사용
- some, every의 특징을 활용하여 break를 사용하는 것처럼 활용 가능
- some -> or 개념
    - 콜백 함수가 true를 반환하면 즉시 순회 중단
- every -> and 개념
    - 콜백 함수가 false를 반환하면 즉시 순회 중단

## reduce
- 배열의 각 요소에 대해 함수 실행, **하나의 결과값을 반환**
    - 원하는 특정 형태의 값으로 변환(숫자, 문자열, 객체, 배열)
- array.reduce(callBackFunction, initialValue)
    - callBackFunction (배열을 처리할 콜백함수)
        - accumulator(acc, 필수)
            - 누적값이며, 이전 콜백 함수가 return 한 값이 다음 순회의 accumulator로 전달
        - currentValue(cur, 필수)
            - 현재 순회에서 처리중인 요소
        - currentIndex(idx, 선택)
            - 현재 처리 중인 currentValue의 인덱스
        - array(arr, 선택): reduce를 호출한 원본 배열
    - initialValue: 누적을 시작할 초기값
