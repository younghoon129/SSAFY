# DB
- 스프레드 시트의 한계
    - 크기

- 데이터베이스 역할
    - 데이터를 저장(구조적 저장)하고 조작
    - 관계형 데이터베이스
        - 데이터 간에 관계가 있는 데이터 항목들의 모음
        - 서로 관련된 데이터 포인터를 저장, 이에 대한 액세스 제공
        - 공통된 id끼리 연결 매칭? 느낌
        - 관계
            - 여러 테이블 간의 (논리적) 연결
            - 데이터를 각각의 테이블에 나눠 저장하되
            - 공통된 키 값을 통해 서로 연결하여 함께 조회, 조작할 수 있음
    
- 관계형 데이터베이스 관련 키워드
    - Table (aka Relation)
        - 데이터를 기록하는 곳
    - Field (aka Column, Attribute)
        - 각 필드에는 고유한 데이터 형식(타입)이 지정됨
    - Record (aka Row, Tuple)
        - 각 레코드에는 구체적인 데이터 값이 저장됨
    - Database (aka Schema)
        - 테이블의 집합
    - Primary Key(기본 키, PK)
        - 각 레코드의 고유한 값
        - 관계형 데이터베이스에서 레코드의 식별자로 활용
    - Foreign Key (외래 키, FK)
        - 테이블의 필드 중 다른 테이블의 레코드를 식별할 수 있는 키
        - 다른 테이블의 기본 키를 참조
        - 각 레코드에서 **서로 다른 테이블 간의 관계**를 만드는 데 사용
    
- RDBMS
    - DBMS
        - 데이터베이스를 관리하는 소프트웨어 프로그램
        - 데이터를 일정한 규칙으로 저장하고
        - 필요할 때 쉽게 꺼내거나 바꿀 수 있도록 도와주는 도구
        - 데이터 저장 및 관리를 용이하게 하는 시스템
        - 데이터베이스와 사용자 간의 인터페이스 역할
        - 사용자가 데이터 구성, 업데이트, 모니터링, 백업, 복구 등 도움
    - RDBMS
        - 종류
            - SQLite
              - 간단하고 효율적인 데이터 저장 및 관리를 제공
              - 설치 없이 가볍게 실행 가능해 모바일 앱, 소규모 프로그램에 적합
            - MySQL
            - PostgreSQL
            - Oracle Database
    - 데이터베이스 정리
        - Table은 데이터가 기록되는 곳
        - Table에는 행에서 고유하게 식별 가능한 기본 키라는 속성 있음
        # 다시
    
- SQL
    - 테이블의 형태로 구조화 된 관계형 데이터베이스에게 요청을 질의(요청)
    - SQL 키워드는 대소문자를 구분하지 않지만 대문자 권장(명시적 구분)
    - 각 SQL Statements 끝에는 세미콜론이 필요
    
- SQL Statements
    - 수행 목적에 따른 SQL Statements 4가지 유형
        - DDL : 구조 및 형식 변경
        - DQL : 데이터 검색
        - DML : 데이터 조작
        - DCL : 사용자 권한 제어
    - DQL
        - SELECT
            - SELECT 키워드 이후 데이터를 선택하려는 필드를 하나 이상 지정
            - FROM 키워드 이후 데이터를 선택하려는 테이블의 이름을 지정
            - 테이블 employees에서 LastName 필드의 모든 데이터를 조회
            - 테이블의 데이터를 조회 및 반환
            - '*' 를 사용하여 모든 필드 선택
    
- Filtering data
    - keywords
        - Clause
            - DISTINCT (중복된 레코드 제거)
            - WHERE
              - 아주 중요함
            - LIMIT
        - Operator
            - BETWEEN
            - IN
            - LIKE
            
- NULL
    - SQL에서 NULL은 실제 값이 아닌 '값이 없음' 또는 '알 수 없음'을 의미
        - 등호(=) 연산자 못씀
        - 등호(=)를 사용하면 어떤 값과도 비교 못하기 때문에 UNKNOWN이 됨
