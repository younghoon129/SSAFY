# CREATE TABLE
- VARCHAR(int) : 가변
    - 뒤에 NOT NULL 하면 NULL값 허용 안함
    
- Constraints
    - 데이터의 무결성을 유지하고 데이터베이스의 일관성을 보장
    - 대표 제약 조건 3가지
        - PRIMARY KEY
            - 해당 필드를 기본 키로 지정
            - INTEGER 타입에만 적용 가능
            - IN, BIGINT 같은 다른 정수 유형은 적용 안됨
        - NOT NULL
            - 해당 필드 NULL 값 허용 안함
        - FOREIGN KEY
            - 다른 테이블의 외래 키 관계를 정의
    
- AUTOINCREMENT
    - 필드의 자동 증가를 나타내는 특수한 키워드
    - 주로 primary key 필드에 적용
    - INTEGER PRIMARY KEY AUTOINCREMENT가 작성된 필드는 
      항상 새로운 레코드에 대해 이전 최대 값보다 큰 값을 할당
    - 삭제된 값은 무시되며 재사용할 수 없게 됨
